What is the Tower Controller?

The Tower Controller is a reference control panel for the UltimateDarkTower library — a software driver for a motorized plastic tower called "Dark Tower" that ships with the board game Return to Dark Tower by Restoration Games.

The tower is a real device that connects to your computer or phone via Bluetooth. It has 24 colored LED lights (at different levels and positions), a speaker, and three rotating drums. The library lets you control all of these features from software.

This test plan walks you through every major feature of the Controller app — connecting, turning lights on and off, playing sounds, rotating the drums, managing game state (glyphs and seals), and monitoring the tower's health (battery, calibration status, dropped skulls). You don't need to know anything about the board game to run these tests.

Before You Start

Browser & Platform Support

This app uses Web Bluetooth to connect to the tower. Supported browsers: Chrome or Edge on desktop, or Chrome on Android. Not supported: Safari (Mac/iOS), Firefox, any browser on iOS (except third-party "Bluefy" app). This is a browser limitation, not the app's fault.

How to Launch the Controller

From the command line in the UltimateDarkTower repo root:

npm install
npm run dev:controller

This builds the app and opens http://localhost:8080 in your browser automatically. If not, open that URL manually in Chrome or Edge.

Choose Your Testing Track

🔴 Real Tower Track

You need: A physical Return to Dark Tower device, powered on, and within Bluetooth range.

Setup: Open the Controller app, leave the "Tower Type" dropdown set to BLE Tower (the default), then click Connect. You'll see your tower listed in the browser's Bluetooth device picker. Select it and pair. The connection banner will turn green: "Tower Connected".

Note: Some steps below are hardware-only (e.g., physically triggered skull drops). Those steps are marked [Hardware Only].

🟦 Tower Emulator Track

You need: Nothing but a modern browser. No physical device required.

Setup: Open the Controller app. Click the Tower Type dropdown and select Tower Emulator. A 3D-rendered tower will pop up in a new window. Click Connect in the main window. You'll connect instantly (no pairing needed).

Tip: This fully simulates the real tower's behavior. All controls work identically.

👉 Pro Tip: You can run both tracks at once — test once with the Emulator (quick feedback loop), then again with real hardware if you have it.

1. Connecting to the Tower

What this checks: The app can detect and pair with a tower (real or emulated), establish a Bluetooth connection, and show connection status.
1.1 Select tower type
Real Tower: Leave the "Tower Type" dropdown set to "BLE Tower" (the default). No action needed.
Emulator: Click the "Tower Type" dropdown at the top of the Controller page and select "Tower Emulator". A new window should pop up with a 3D tower model inside.
Expected: Emulator track — a popup appears titled "Tower Emulator" with a 3D tower visible. Real track — no visible change to the main app.
1.2 Click Connect
Real Tower: Click the green Connect button. A browser dialog will appear listing nearby Bluetooth devices. Find your tower (device name is "ReturnToDarkTower") and click it to pair.
Emulator: Click the green Connect button. Connection should happen instantly (no device picker).
Expected: The connection state banner at the top (near "Connect" button) turns green and displays "Tower Connected". The Disconnect button becomes enabled. Depending on device, you may see the 3D emulator tower light up or respond to show it's connected.
1.3 Verify connection state
Look at the right side of the page. You should see status cards for Battery (with a percentage), Calibration (with three status icons for Top/Middle/Bottom drums), and Skulls (a counter). The Calibration icons may show red X (not yet calibrated) or green check marks (calibrated).
Expected: Battery percentage appears (e.g., "85%"). Calibration icons are visible (likely showing red X for all three drums if this is the first connection). The Skulls counter shows "0" or some number. If any of these are missing or show "—" / "?", the connection is not working properly.

2. Disconnecting

What this checks: The app can cleanly disconnect from the tower and show disconnected state.
2.1 Click Disconnect
Find the red Disconnect button (next to the Connect button) and click it.
Expected: The connection state banner turns red and displays "Tower Disconnected". The Connect button becomes enabled. The Disconnect button becomes disabled. Battery/Calibration/Skulls cards may show "—" or "?" to indicate unknown state.
2.2 Reconnect
Click Connect again. (Follow the same pairing flow as step 1.2 if using a real tower; Emulator reconnects instantly.)
Expected: Connection state banner turns green "Tower Connected" again. Status cards repopulate with battery/calibration/skulls.

3. Calibration

What this checks: The app can initiate a calibration routine, which teaches the tower the reference position of its three drums (required before rotation commands work reliably).
3.1 View calibration status
Look at the Calibration status card on the right side. It shows three small icons — one each for Top, Middle, and Bottom drums.
Expected: If not yet calibrated, all three show a red X (❌). If already calibrated, they show green check marks (✓). Hover over any icon to see a tooltip confirming "Top Drum", "Middle Drum", or "Bottom Drum".
3.2 Start calibration
Real Tower: Click the blue Calibrate button (in the Diagnostics tab). The tower will begin spinning its drums to find their zero positions. This takes about 10-15 seconds. Do not interrupt the tower during this time.
Emulator: Click the blue Calibrate button. In the emulator popup, the 3D tower will visually spin. Calibration completes almost instantly in the emulator.
Expected: A message appears near the Calibrate button saying "Calibrating...". The Calibrate button becomes disabled (grayed out). The 3D emulator (if using Emulator track) shows the drums spinning.
3.3 Verify calibration complete
Wait for the calibration to complete. The "Calibrating..." message should disappear, and the Calibrate button should become enabled again.
Expected: All three calibration icons turn green (✓). The message "Calibrating..." disappears. The Calibrate button is enabled again and clickable.

4. Lights

What this checks: The app can control the tower's 24 LEDs individually and in groups, apply different lighting effects (on, breathe, flicker), and turn all lights on or off.
4.1 Turn on a single light
Scroll down to the "Lights" section on the page. You'll see a grid of checkboxes labeled "Doorway" (12 lights: 4 sides × 3 levels), "Ledge" (4 corner lights), and "Base" (8 lights). Click on one of the checkboxes to select a light.
Expected: The checkbox is now checked. In the 3D emulator (if using Emulator track), that light LED should turn on. On a real tower, the physical LED lights up. The light state is saved in the app's status display (the "Status Packet" grid at the bottom).
4.2 Turn off the light
Click the same checkbox again to uncheck it.
Expected: The checkbox unchecks. The LED turns off (in the emulator, the light dims or turns dark; on real hardware, the physical light turns off).
4.3 Turn on all lights
Above the light grids, find the blue All On button and click it.
Expected: All 24 light checkboxes become checked. On the real tower or in the emulator, all LEDs turn on (solid color, not blinking/breathing).
4.4 Apply a light effect
Before step 4.5, ensure all lights are on from step 4.3. Then, above the light grids, find the Light Style dropdown and select "Breathe" (a pulsing/breathing effect). Click the blue Apply to All Lights button.
Expected: All lights begin pulsing/breathing (brightening and dimming in a cycle). In the emulator, you should visually see the lights pulse. On real hardware, the physical LEDs pulse.
4.5 Turn off all lights
Find the blue All Off button and click it.
Expected: All 24 light checkboxes become unchecked. All LEDs turn off (emulator or real tower).

5. Sound & Volume

What this checks: The app can select and play sound effects from the tower's built-in sound library, and adjust the tower's speaker volume.
5.1 Select a sound
In the Diagnostics tab, find the Sound dropdown. Click it to see the list of available sound effects (e.g., "Glyph Reveal", "Tower Laugh", "Seal Break", etc.).
Expected: The dropdown opens and displays a list of sound names. Select any sound (e.g., "Glyph Reveal").
5.2 Play the sound
Real Tower: Click the blue Play button next to the Sound dropdown. The tower's speaker will play the selected sound effect.
Emulator: Click the blue Play button. A sound effect will play from your computer's speakers (the emulator relays the audio).
Expected: A sound plays from the tower (or your computer if using Emulator). The sound should be distinct and audible. The app may log "Sound played: [sound name]" in the Log panel on the right.
5.3 Adjust volume
In the Diagnostics tab, find the volume control: two buttons labeled "<" (volume down) and ">" (volume up). Next to them is a text label showing the current volume level (e.g., "Loud", "Medium", "Quiet"). Click the volume down button (<) a few times to lower the volume.
Expected: The volume label changes (e.g., from "Loud" to "Medium" to "Quiet"). An emoji icon next to it also changes (speaker with lines → speaker with one line → speaker with X, etc.) to visually show the volume level. The next sound you play should be quieter.

6. Drum Rotation

What this checks: The app can rotate the tower's three drums to specific compass positions (North, South, East, West). The tower has three independent drums (Top, Middle, Bottom) that must all be calibrated before rotation works reliably.
6.1 Select drum positions
In the Diagnostics tab, scroll down to the "Drums / Rotation" section. You'll see three dropdowns labeled "Top", "Middle", and "Bottom". Each dropdown has four options: North (N), South (S), East (E), West (W). Set each dropdown to a different compass direction (e.g., Top = North, Middle = East, Bottom = South).
Expected: Each dropdown displays your chosen direction. No changes to the tower yet — you're just selecting the target positions.
6.2 Rotate the drums
Real Tower: Click the blue Rotate button below the three drum dropdowns. The tower's drums will spin to their target positions. This takes a few seconds. The tower may make clicking/whirring sounds as the motors move.
Emulator: Click the blue Rotate button. The 3D tower in the emulator popup will visually spin its drums to face the chosen directions.
Expected: The drums rotate and settle into place facing the chosen compass directions. (Real tower: you may hear mechanical sounds; Emulator: you see the 3D drums spin). The Status Packet display at the bottom updates to show the new drum positions.

7. Randomizing Drums

What this checks: The app can rotate one or more drums to random compass directions in a single command.
7.1 Choose drums to randomize
Click the Library tab to see a different set of controls. Scroll down to the "Randomize Drums" section. Find the Level(s) dropdown and select an option (e.g., "All" to randomize all three drums, or "Top" to randomize only the top drum).
Expected: The dropdown displays your selected option. No changes to the tower yet.
7.2 Randomize
Click the blue Randomize button below the Level(s) dropdown.
Expected: The selected drum(s) spin to random compass directions. The Status Packet at the bottom updates to show the new random positions. (If you chose "All", all three drums should move; if you chose "Top", only the top drum moves while middle and bottom stay in place.)

8. Glyphs

What this checks: The app can track and manipulate glyphs — symbols drawn on the tower's drums that face different compass directions based on drum rotation. Glyphs represent in-game concepts (Cleanse, Quest, Battle, Banner, Reinforce).
8.1 View glyph positions
In the Library tab, scroll down to the "Glyphs" section. You'll see a 3×4 grid labeled with drum levels (Top / Mid / Bot rows) and compass directions (N / E / S / W columns). Below the grid is a dropdown to select a glyph (Cleanse, Quest, Battle, Banner, or Reinforce) and another dropdown to select a compass direction.
Expected: The grid displays. If any glyphs are currently active in their positions, those grid cells glow (amber/yellow highlight). Otherwise, all cells are dark/empty.
8.2 Move a glyph to a position
In the Glyphs section, select a glyph from the first dropdown (e.g., "Cleanse") and a compass direction from the second dropdown (e.g., "North"). Then click the blue Move Glyph button.
Expected: The tower's drums rotate so that the chosen glyph faces the chosen compass direction. The glyph-position grid updates to show the glyph now occupies that spot (one cell glows amber). The status display updates to show the new drum positions that put this glyph in place.

9. Seals

What this checks: The app can track and manage seals — physical mechanisms in the tower at each of the 12 light positions (3 levels × 4 sides). Breaking a seal triggers a visual effect (lights flash) and updates game state. Seals can also be cleared/reset.
9.1 View seal status grid
Scroll down to the "Seals" section in the Library tab. You'll see a 3×4 grid of square buttons (representing the 12 seal positions: 3 levels × 4 compass directions). A dropdown above the grid lets you select a seal location (e.g., "North Top", "South Mid", etc.).
Expected: The grid displays 12 empty (gray or white) squares. None are currently broken.
9.2 Break a seal via dropdown
Real Tower: Select a seal location from the dropdown (e.g., "North Top"). Click the blue Break Seal button. The tower will trigger a seal-break effect: lights at that position may flash, and a sound may play.
Emulator: Select a seal location from the dropdown. Click the blue Break Seal button. The emulator popup will show a visual seal-break effect (the seal at that position drops or disappears, and lights flash).
Expected: The corresponding grid square turns red/dark red (indicating the seal is now broken). The button may be disabled for a brief moment to prevent rapid repeated breaks. The Log panel records "Seal broken: [location]".
9.3 Break a seal via grid click
Click directly on one of the gray (unbroken) seal squares in the grid (a square that is not red).
Expected: That square turns red (broken). A seal-break effect triggers on the tower/emulator. The dropdown updates to show the location you just clicked.
9.4 Clear all seals
Click the blue Clear Effect button (below the seal grid).
Expected: All grid squares turn back to gray (not broken). All lights on the tower turn off. The seal state is reset. The dropdown resets to "select a location". The Log records "Seals cleared".

10. Battery & Skulls

What this checks: The app displays the tower's current battery percentage and tracks the count of skulls (tokens) that have been dropped from the tower during gameplay.
10.1 View battery status
Look at the right side of the page. Find the Battery card (shows a battery icon). It displays a percentage (e.g., "85%") and a trend arrow (⬆️ increasing, ⬇️ decreasing, or ➡️ stable) showing whether battery level is rising, falling, or holding steady.
Expected: A battery percentage is visible. For Emulator, this is simulated and stable. For real tower, this reflects the actual tower's battery voltage.
10.2 View skull counter
Find the Skulls card on the right side (shows a skull icon 💀). It displays a number (e.g., "0") representing how many skull tokens have been dropped from the tower. Next to the number is a small icon button to reset the counter.
Expected: The skull counter is visible and displays a number (usually 0 if no skulls have dropped yet).
10.3 Reset skull counter [Hardware Only]
[Note: This step requires a real tower with skulls already dropped. Emulator does not simulate physical skull drops.] Click the small icon button next to the skull counter to reset it to 0.
Expected: The skull counter resets to 0. The Log records "Skull count reset".

11. Troubleshooting Modal

What this checks: The app can display a help/troubleshooting guide for common issues (tower jams, disconnects, firmware updates, battery specs).
11.1 Open Troubleshooting modal
Scroll to the bottom of the page. Find the Troubleshooting button (usually in a footer or below the main content).
Expected: The button is visible and clickable.
11.2 Modal displays content
Click the Troubleshooting button. A modal popup should appear with troubleshooting content.
Expected: A modal dialog opens overlaying the page. It displays troubleshooting sections (e.g., "Tower Jam Recovery", "Tower Disconnects During Play", "Firmware Update Failures", "Battery Specifications") with readable text/instructions.
11.3 Close modal
Close the modal by clicking the X button in the top-right corner, clicking outside the modal, or pressing Escape on your keyboard.
Expected: The modal closes and you're back at the Controller page.
