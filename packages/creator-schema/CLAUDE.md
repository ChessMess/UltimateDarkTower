# packages/creator-schema (`@udtc/schema`) — canonical scenario schema

The canonical JSON Schema for Creator scenarios. Part of the private `@udtc/*` subsystem.

- **`src/scenario.schema.json` (JSON Schema draft 2020) is the single source of truth** for
  the `node.kind` enum and the closed per-kind property set. Its versioning changelog lives
  inline in the schema's `$comment` fields.
- `src/index.ts` exports `scenarioSchema` (raw JSON) and `validateL1` — the Ajv-compiled
  (lazily memoized) L1 validator. This is the L1 tier; L2/L3 validation lives in
  `packages/creator-adapters`.
- `src/types.ts` is marked "Generated by json-schema-to-typescript — do not hand-edit" but
  is currently a **`// TODO: Phase 1` stub**; the generation step isn't wired up yet.

## 0.5.0 is the first non-backward-compatible schema change

Every 0.4.x bump kept the promise that "every earlier document validates unchanged." **0.5.0
breaks it on purpose**: `$defs/boardDef.anchors` (the closed five-key building/skull/hero/foe/marker
slot map) is _replaced_ by `spots` (an open list of `{id, at, accepts}`), so a document authored
before 0.5.0 no longer validates — there is no `oneOf`/migration branch. `library.tokenTypes` also
gains optional presentational fields (`artRef`/`color`/`capacity`) so an entry can double as a
board-renderable type. Consumers (`apps/creator`, `apps/digital`, `apps/player`) detect the older
`schemaVersion` at their own load boundary and offer an export-then-clear dialog instead of
migrating — see `docs/creator/board-designer.md` and `packages/board/docs/STATE_MODEL.md`'s
"Old data: refuse, don't migrate".

## Build & test

`build` is a no-op echo; `typecheck` = `tsc --noEmit`. **Test is plain Node**, not vitest:
`node test/conformance_test.js` (a hand-rolled `check()` pass/fail counter that first
Ajv-compiles the schema in `strict` mode — a compile failure hard-fails the run). No
`@udtc/*` deps — it's a leaf. This kind set must stay in sync with `apps/creator`'s
`NODE_KINDS` and `creator-engine`'s `EngineNode` (root `pnpm validate:nodes` checks this).
